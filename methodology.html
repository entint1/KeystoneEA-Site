Pillar 03 — Methodology

From first catalog to strategic driver.

The metamodel and viewpoints describe what you model. The methodology describes how you get there — a staged path that takes an EA practice from an empty repository to an AI-powered function embedded in business planning.

Two tracks run in parallel

Every phase advances on two fronts at once. The Content track builds the substance stakeholders consume — catalogs, capability models, portfolio assessments. The Architecture Readiness track keeps the practice itself fit — model quality, governance, viewpoints, and the AI service portal. Select a phase to explore both.
