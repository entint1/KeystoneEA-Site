What it is

A single connected model of the enterprise.

Most enterprise architecture frameworks are either too heavy to maintain or too loose to trust. Keystone EA™ is built for the middle: rigorous enough to keep a live repository consistent, light enough that a small team can actually run it.

It describes an organisation — its strategy, governance, business, data, applications and technology — as one model, where every element has a defined type and every link is governed by an explicit rule. From a board-level driver down to a database engine, everything traces through the same fabric.

The standard rests on three parts that fit together: a metamodel, a viewpoint library, and an implementation methodology.

The layered model

Seven layers, top to bottom.

Keystone EA™ stacks the enterprise into seven layers. Upper layers express intent; lower layers deliver it. Every object type lives in exactly one layer.

Design principles

Opinionated where it counts.

Lean by design

Nineteen object types — not hundreds. Enough to model any enterprise faithfully, few enough that the model stays readable and a small team can maintain it.

Any-to-any, with rules

A universal Connection means nothing is ever un-modelable, while a complete rules matrix governs the qualified relationships — so the model stays both flexible and consistent.

Viewpoint-driven

You never face the whole model at once. Viewpoints filter it down to the handful of object types that answer one stakeholder's question — governance, strategy, capability or integration.

AI-native delivery

The methodology assumes a live repository accelerated by AI — from structuring an AI service portal early to autonomous impact analysis and predictive insight at maturity.

Traceable end to end

Realization links thread a single line of sight from a strategic driver, through capabilities and applications, down to the infrastructure that ultimately delivers it.

Governed and current

Principals and standards are first-class objects, so accountability and compliance are modelled — not bolted on — and the technology reference model is wired into governance.
