Pillar 02 — Viewpoints

One model, many perspectives.

A full enterprise model is too dense to read all at once. A viewpoint is a saved lens — it narrows the model to just the object types that answer one stakeholder's question. Keystone EA™ ships with six, plus the all-inclusive master view.

Viewpoints are subsets, not silos

Every viewpoint draws from the same underlying model. An application that appears in the Capability Map is the very same object you'll find in Technology Usage — so insight gathered through one lens stays consistent with every other. The master viewpoint simply removes the filter.
